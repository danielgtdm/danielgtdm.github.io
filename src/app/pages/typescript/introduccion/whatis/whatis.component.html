¿Qué es JavaScript?

No podemos entender TypeScript sin antes entender JavaScript; sería como querer correr antes de aprender a caminar.

JavaScript es un lenguaje de programación dinámico y muy flexible, el cual es considerado el lenguaje universal de la web; es decir, todo programa usado en la web debe hablar JavaScript o nadie hablará con él.

JavaScript llegó al mundo como el lenguaje del lado del cliente y así fue durante un buen tiempo hasta la llegada de Node.js, un entorno de ejecución del lado del servidor que habla en JavaScript. Desde entonces, también es posible usarlo en el servidor.

Es entonces cuando se comienzan a hacer aplicaciones del lado del servidor con Node.js. ¿Qué mejor que usar el mismo lenguaje del front-end en el back-end, verdad? Ya no necesitas tener un programador JavaScript y otro programador en Java. Con JavaScript ya puedes hacer de todo, ¿verdad? ¿eh?

Justo aquí es cuando el tipado débil de JavaScript comienza a ser un problema. A medida que el proyecto crece, se vuelve más difícil mantener el control y el orden, y encontrar errores se convierte en todo un desafío. Por lo tanto, hacer aplicaciones escalables no es un lujo que te puedas dar con facilidad.

¿Eh? Ahora ya no parece ser tan buena idea hacer todo con JavaScript, sobre todo si hablamos de proyectos grandes, ¿verdad?

Es por esta raz por la que a JavaScript le ha costado ganar terreno en el lado del servidor, teniendo que competir con Java, C#, e incluso PHP.
