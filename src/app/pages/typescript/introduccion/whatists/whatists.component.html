¿Qué es TypeScript?

Podriamos decir que TypeScript es una forma de escribir JavaScript, pero de una manera mucho más estandarizada y controlada, a lo que denominamos un lenguaje fuertemente tipado. En pocas palabras, TypeScript es JavaScript con esteroides.

TypeScript se orienta hacia la programación orientada a objetos, lo que ofrece una estructuración de datos en la cual podemos utilizar clases, herencia e interfaces. Esto es muy conveniente en proyectos extensos, ya que nos permite trabajar con ciertos patrones de diseño.

Normalmente se dice que TypeScript es un lenguaje compilado, pero el término correcto es "transpilado". Esto se debe a que todo lo que escribimos en TypeScript no se traduce al lenguaje máquina, sino que se convierte a JavaScript.

Creo que popularmente se le dice que es compilado debido a que en otros lenguajes populares como Java o C++ se usa ese término, en los cuales está correctamente implementado, porque su compilador traduce las instrucciones (código fuente) al lenguaje máquina. Sin embargo, este no es el caso de TypeScript. Además, al llamar a su “compilador” de esta manera aporta a que lo llamen lenguaje compilado.

Para evitar discusiones, en este blog seguiré utilizando el término "compilado" para referirme a TypeScript, ya que es la forma en que comúnmente se le nombra, y no tengo la intención de cambiar eso.

Partes de TypeScript

TypeScript se compone principalmente de 3 partes:

  1. Lenguaje: Incluye la sintaxis, los tipos, las palabras reservadas, entre otros.

  2. TypeScript Compiler (TSC): Este es el compilador que traduce el código fuente en TypeScript a su equivalente en JavaScript.

  3. TypeScript Languaje Service: Este es un conjunto de herramientas y API que permite a los editores de texto, como Visual Studio Code, analizar tu código mientras escribes y ofrecerte funciones como autocompletado, refactorización y navegación en el código, entre otras.
